# Coding Rules

**Version Control:** Every time we successfully complete a page or a major feature from `site-structure.md`, and I say "looks good" or approve the result, you must automatically run a Git commit. Write a concise, professional commit message summarizing what we built before asking me what to do next.

**Tech Stack & Architecture:**
Strictly utilize Vanilla Web Native technologies (HTML5, CSS3, vanilla JavaScript). Do not use any frontend frameworks (React, Vue), CSS preprocessors (Sass), or Static Site Generators. The output must be raw, static files optimized for direct deployment to GitHub Pages (`.github.io`).
